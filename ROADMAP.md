# Suggested Next Topics

The tutorial now covers a solid beginner foundation. If you want to keep extending it, these topics would make the next best additions.

## Beginner topics to add next

1. **User input and validation**
   - Prompt users for values.
   - Convert input strings to numbers.
   - Validate input before using it.

2. **For loops and ranges**
   - Loop over lists, strings, dictionaries, and `range()` values.
   - Compare `for` loops with the existing `while` loop examples.

3. **List methods and common operations**
   - Use `append`, `remove`, `pop`, `sort`, and `len`.
   - Search lists with `in`.
   - Combine list methods with loops.

4. **Function parameters and return values**
   - Pass values into functions.
   - Return calculated results.
   - Reuse functions in larger examples.

5. **Boolean logic**
   - Combine conditions with `and`, `or`, and `not`.
   - Use boolean values in validation and menu programs.

6. **Working with packages and virtual environments**
   - Explain why projects use virtual environments.
   - Install a small package with `pip`.
   - Import and use an installed package.

7. **Testing basics**
   - Add simple `assert` examples.
   - Introduce `unittest` or `pytest`.
   - Show how tests help learners change code safely.

8. **Debugging basics**
   - Read tracebacks.
   - Use `print` debugging.
   - Introduce breakpoints at a beginner level.

9. **Small projects**
   - Build a calculator.
   - Build a number guessing game.
   - Build a grade book.
   - Build a todo list that saves to a file.

## Intermediate topics for later

1. **Object-oriented programming practice**
   - Add multiple classes.
   - Show class relationships.
   - Demonstrate inheritance only after learners understand basic classes.

2. **CSV and JSON files**
   - Read and write structured data.
   - Compare plain text, CSV, and JSON.

3. **Command-line arguments**
   - Use `sys.argv` or `argparse`.
   - Turn lesson examples into reusable scripts.

4. **Type hints**
   - Add annotations to variables and functions.
   - Explain how type hints help readability.

5. **Project organization**
   - Split code into multiple files.
   - Use a `main()` function.
   - Explain imports between local files.

6. **APIs and web requests**
   - Fetch data from a public API.
   - Parse JSON responses.
   - Handle network errors.

7. **Databases**
   - Introduce SQLite.
   - Store and query simple records.

## Recommended order

If you only add a few more lessons, add them in this order:

1. `16ForLoops.py`
2. `17InputValidation.py`
3. `18FunctionReturns.py`
4. `19ListMethods.py`
5. `20Testing.py`
6. `21MiniProject.py`
